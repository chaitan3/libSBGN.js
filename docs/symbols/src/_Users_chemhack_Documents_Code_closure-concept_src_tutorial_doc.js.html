  3  *<h1>General Information of libSBGN.js</h1>
  4  *
  5  * <p>In general, libSBGN.js is modeled from SBGN(System Biology Graph Notation). All classes and functions of libSBGN.js is prefixed with “sb.”. A system biology graph is represented as a sb.Document. sb.Document contains instances of sb.Node and sb.Arc as well other objects like sb.ArcGroup and sb.Port. You can have a look at the API reference.</p>
  6  *
  7  *<h1>Tutorial 1: Operating sb.Document, sb.Node</h1>
  8  *
  9  *<p>var doc=new sb.Document(); //This create new sb.Document.
 10  *var node=doc.createNode(‘node1’).type(sb.NodeType.SimpleChemical).label(‘ATP’); //This create a node with id ‘node1’ as a simple chemical with label text ‘ATP’
 11  *node.label() //Returns ‘ATP’
 12  *node.id() //Returns ‘node1’
 13  *node.attr(‘cssClass’,‘highlight’) //Custom attributes could be accessed via attr() method.
 14  *node.attr(‘cssClass’) //Returns ‘highlight’
 15  *doc.node(‘node1’).label() //Returns ‘ATP’. Nodes can also be accessed from sb.Document with its id.</p>
 16  *

 20  *

 23  *doc.createNode(‘node1’)</p>

 25  *<h1>Tutorial, reading a file</h1>
 26  *
 27  *<p>Currently only SBGN-ML is supported by the file reader. But our goal is to support jSBGN, SBGN-ML, SBML and BioPAX. You can either read a file from a string object which contains the file content or a URL points to the file on the internet. If you use the URL, the file will go through our proxy which wraps the file content with jsonp(to overcome cross domain HTTP request restrictions of web browsers). We will not either log, store, or share the data go through our proxy. And you can easily setup your own proxy as well.</p>
